# 001-xxx: Flovia Agent Offers MVP Implementation Plan

## Goal

`docs/SPEC.md` の MVP demo flow を実装する。

優先する完成状態:

- Merchant API が personalized HTTP 402 を返す。
- Flovia quote API が rule-based offer を返す。
- demo-agent が 402 の `flovia` field を読み、MVP simulation payment で retry できる。
- dashboard API が request / offer / conversion / revenue / segment を返す。

## Current Repository State

現状は Bun workspace starter に近く、SPEC の monorepo 構成は未整備。

既存要素:

- `docs/SPEC.md`
- root `package.json`
- root `tsconfig.json`
- starter の `packages/example`

## Implementation Strategy

最小実装で demo path を先に通す。Privy / x402 / Postgres の本番連携は後続フェーズに残し、MVP では explicit simulation と in-memory store を使う。

実装順:

1. Bun workspace を `apps/*` + `packages/*` に変更する。
2. `packages/shared` に Zod schema と型を置き、API / SDK / agent の契約を固定する。
3. `packages/offer-engine` に rule-based `computeOffer` を実装する。
4. `packages/db` は Drizzle の前段として in-memory repository を実装し、dashboard seed data を返せるようにする。
5. `apps/api` に Hono backend を実装する。
6. `packages/sdk` に Hono middleware `floviaAdaptive402` を実装する。
7. `apps/merchant-api` に demo x402-gated endpoints を実装する。
8. `apps/demo-agent` に CLI demo runner を実装する。
9. `packages/ui` と `apps/web` は demo API flow が安定してから最小 UI として追加する。

## MVP Scope Decisions

今回実装するもの:

- `POST /v1/offers/quote`
- `POST /v1/events/request`
- `POST /v1/events/payment`
- `GET /v1/merchants/:merchantId/dashboard`
- anonymous wallet vs Privy-authorized simulation の価格差
- first call discount / repeat loyalty / low-budget alternative / premium upsell の rule-based logic
- merchant-side HTTP 402 response with `accepts[0]` and `flovia`
- `X-PAYMENT` JSON header による explicit MVP simulation mode

今回は本番実装しないもの:

- real Privy login / embedded wallet UI
- real x402 facilitator payment verification
- Drizzle + Postgres persistence
- Next.js dashboard UI の完成版
- cross-app Privy identity lookup

## Privacy And Security Constraints

Merchant に raw Privy / PII を渡さない。MVP simulation でも merchant response に含めるのは以下だけにする。

- normalized buyer source
- segment
- agent score
- policy
- reason codes
- offer decision

## Verification Plan

最小確認:

- `bun install`
- `bun run typecheck`
- `bun run dev:api`
- `bun run dev:merchant`
- `bun run dev:agent -- --budget 0.25 --privy-authorized`

期待結果:

- agent の初回 request は HTTP 402 を受ける。
- 402 response に `flovia.finalPrice` と `accepts[0].amount` が一致している。
- Privy-authorized simulation では base price より安い offer になる。
- retry request は paid API response を受ける。
- dashboard endpoint は seeded data または recorded event data を返す。
